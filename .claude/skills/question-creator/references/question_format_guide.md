# 题目格式完整指南

## 目录结构

### 基准测试题库

```
{题库名}/
├── base-test/
│   └── NNN-problem-name/
├── advanced-test/
│   └── NNN-problem-name/
├── final-test/
│   └── NNN-problem-name/
└── final-test+/
    └── NNN-problem-name/
```

### 题库名称映射

| 类别 | 目录名 | category 值 |
|------|--------|-------------|
| 代码能力 | 代码能力基准测试题库 | code |
| 数理能力 | 数理能力基准测试题库 | math |
| 逻辑能力 | 自然语言与逻辑能力基准测试题库 | logic |
| 综合能力 | 综合能力测评 | comprehensive |

### 难度映射

| 难度 | 目录名 | difficulty 值 |
|------|--------|---------------|
| 基础 | base-test | base |
| 进阶 | advanced-test | advanced |
| 终极 | final-test | final |
| 终极+ | final-test+ | final+ |

## meta.yaml 格式

```yaml
id: {category}-{difficulty}-{number}
brief: 题目简短描述（一句话）
category: math | code | logic | comprehensive
difficulty: base | advanced | final | final+
scoring_std:
  max_score: 10
  indicators:
    - indicator1
    - indicator2
```

### id 命名规则

格式：`{category}-{difficulty}-{number}`

示例：
- 数理题：`math-base-001`, `math-adv-002`, `math-final-001`
- 代码题：`code-base-001`, `code-adv-001`, `code-final-001`
- 逻辑题：`logic-base-001`, `logic-adv-001`, `logic-final-001`
- 综合题：`comp-001`, `comp-002`

### 可用评分指标

#### 代码类 (code)

| 指标 | 说明 |
|------|------|
| `ans_correct` | 答案正确性 - 代码能否正确运行并得到预期结果 |
| `code_quality` | 代码质量 - 代码风格、注释、可读性 |
| `efficiency` | 运行效率 - 算法时间和空间复杂度 |
| `robustness` | 鲁棒性 - 异常处理和边界情况考虑 |

#### 理论类 (theory)

| 指标 | 说明 |
|------|------|
| `completeness` | 回答完整性 - 是否涵盖问题的所有方面 |
| `accuracy` | 准确性 - 概念和信息是否准确无误 |
| `clarity` | 表达清晰度 - 逻辑是否清晰、表述是否易懂 |
| `depth` | 深度 - 回答是否有深度和见解 |

#### 设计类 (design)

| 指标 | 说明 |
|------|------|
| `ans_correct` | 答案正确性 - 设计方案是否正确 |
| `example_quality` | 示例质量 - 举例是否恰当、清晰 |
| `completeness` | 回答完整性 - 是否考虑了各种场景 |
| `practicality` | 实用性 - 方案是否具有实际应用价值 |

## prompt.md 格式

纯净的 prompt，直接发给被测模型。不包含任何元数据或格式说明。

### 数理题示例

```markdown
鸡兔同笼，共35只头，94只脚，问鸡兔各多少？

请给出完整的解题过程和最终答案。
```

### 代码题示例

```markdown
创建一个简单的计算器网页应用，要求：
1. 支持基本四则运算（加、减、乘、除）
2. 包含数字0-9按钮、小数点、运算符按钮和等号
3. 显示区域展示当前输入和计算结果
4. 支持连续计算

仅返回完整的HTML代码，不包含markdown格式标记。
```

### 逻辑题示例

```markdown
有两个盒子A和B，其中一个装着糖果，另一个是空的。盒子上分别贴着纸条：
- A盒："糖果在B盒里"
- B盒："两个盒子的纸条都在撒谎"

已知**只有一张纸条说的是真话**，请问糖果在哪个盒子里？
```
